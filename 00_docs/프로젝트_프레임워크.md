# WEAV-AI 프로젝트 프레임워크·구성

프로젝트의 기본 프레임워크와 디렉터리 구조를 정리한 문서입니다.

---

## 1. 프론트엔드

| 구분 | 기술 |
|------|------|
| **런타임/빌드** | Node.js, Vite 6 |
| **언어** | TypeScript 5.6 |
| **UI 프레임워크** | React 19 |
| **스타일** | Tailwind CSS 3, PostCSS, Autoprefixer |
| **아이콘** | lucide-react |
| **마크다운** | react-markdown, remark-gfm |
| **코드 하이라이트** | react-syntax-highlighter |
| **라우팅** | 없음 (단일 화면, React Router 미사용). 세션 kind(chat/image/studio)로 화면 전환 |
| **상태** | React Context (ChatContext) |
| **HTTP** | fetch 기반 `apiClient` (get/post/patch/delete) |
| **문서 뷰어** | pdfjs-dist (PDF 렌더링 + citations 하이라이트) |
| **진입점** | index.html → index.tsx → App.tsx |

### 디렉터리 구조

- `src/components/chat/` — ChatView, ChatInput, ChatMessage, ModelSelector
- `src/components/studio/` — StudioView (WEAV Studio: 기획·대본·이미지·음성·영상 생성)
- `src/components/layout/` — AppHeader, Sidebar
- `src/contexts/` — AppContext, ChatContext, ToastContext, LayoutContext
- `src/services/api/` — apiClient, sessionApi, chatApi
- `src/services/studio/` — studioFalApi, geminiService, trendingApi
- `src/constants/` — models.ts
- `src/types.ts`, `src/index.css`

### 개발

- `npm run dev` → Vite dev (기본 포트 3001), `/api` 프록시 → localhost:8080
- ESLint, `tsc --noEmit` (Prettier 미설정)

## 2. 백엔드

| 구분 | 기술 |
|------|------|
| **언어** | Python 3.12 (Dockerfile 기준) |
| **프레임워크** | Django 4.2 |
| **API** | Django REST Framework 3.14 |
| **DB** | PostgreSQL + pgvector (Docker 기본). `DATABASE_URL`이 없으면 SQLite로 폴백 |
| **설정** | python-decouple, python-dotenv |
| **CORS** | django-cors-headers |
| **비동기** | Celery + Redis (worker 분리) |
| **스토리지** | MinIO (S3 호환, presigned URL) |
| **문서 처리** | PyMuPDF(텍스트/bbox), (선택) pytesseract OCR, fal(VLM) 이미지 OCR, LibreOffice(HWP/HWPX->PDF) |
| **AI 연동** | fal.ai HTTP API(OpenRouter router 포함) + Pydantic 2 |
| **Studio** | Google AI Studio Gemini API(유튜브 벤치마킹) + YouTube API(트렌딩) |
| **WSGI** | Gunicorn 21 |

### Django 앱

- **users** — AUTH_USER_MODEL, admin (기본만)
- **ai** — 채팅·이미지·Job API (`/api/v1/chat/...`)
- **chats** — 세션/메시지/문서 업로드 + RAG 색인 (`/api/v1/sessions/...`)
- **core** — 헬스체크 + WEAV Studio/유튜브 API (`/api/v1/health/`, `/api/v1/studio/...`)
- **jobs** — (레거시/확장) PDF 처리 관련 Celery task 모음

### core(Studio) 요약

- `apps/core/views.py` — health, youtube_trending, studio_llm, studio_image, studio_tts, studio_video, studio_export 등
- `apps/core/urls.py` — `/health/`, `/studio/*` 라우팅
### apps.ai 내부

- `views.py` — complete_chat, complete_image
- `router.py` — 텍스트/이미지 라우팅 (OpenRouter·fal.ai)
- `fal_client.py` — fal.run HTTP 호출
- `schemas.py` — TextGenerationRequest, ImageGenerationRequest (Pydantic)
- `errors.py` — AI 관련 예외

### 문서 업로드 + RAG(요약)

- 업로드: `apps/chats/views.py::session_upload` → MinIO 업로드 + Document 레코드 생성 + Celery `process_pdf_document` enqueue
- 색인: `apps/chats/tasks.py::process_pdf_document`
  - PDF 텍스트/bbox 추출 + 이미지 추출 + OCR(선택) → `ChatMemory(pgvector)`에 저장
- 사용: 채팅 프롬프트에서 `@문서명` 멘션 시 해당 문서만 검색하고 citations(bbox/page)을 내려줘 프론트에서 하이라이트

### 실행

- `manage.py`, `config/wsgi.py`, `config/asgi.py`
- Docker: `Dockerfile` + `entrypoint.sh` (DB 대기 → migrate → collectstatic → Gunicorn)

## 3. 인프라 (infra)

| 구분 | 기술 |
|------|------|
| **오케스트레이션** | Docker Compose |
| **서비스** | postgres(pgvector/pg15), redis (Redis 7), minio, api (Django/Gunicorn), worker (Celery), nginx (Alpine) |
| **리버스 프록시** | Nginx — 포트 8080 → api:8000, `/api/`, `/admin/`, `/healthz` |
| **네트워크** | weavai_network (bridge) |
| **볼륨** | postgres_data, minio_data (영속) |

### Nginx

- `infra/nginx/conf.d/weavai.conf` — API·admin·healthz 라우팅, limit_req_zone 기반 rate limit (30r/m, burst 10)
- (선택) `infra/nginx/conf.d/admin_allowlist.conf` — admin IP 제한

## 6. 개발 환경 (OS)

- **macOS / Linux**: Makefile (`make up`, `make build` 등) 사용.
- **Windows**: Docker Desktop 설치 후, 프로젝트 루트에서 `compose.ps1`(PowerShell) 또는 `compose.cmd`(CMD)로 동일 명령 실행. `.gitattributes`로 셸 스크립트(`*.sh`)는 LF 줄바꿈 유지해 Docker 내부 실행이 안정적.

## 7. 한 줄 요약

- **프론트**: React 19 + TypeScript + Vite 6 + Tailwind, 채팅/이미지 UI + WEAV Studio, pdfjs 기반 문서 근거 하이라이트.
- **백엔드**: Django 4 + DRF + PostgreSQL(pgvector) + Celery/Redis + MinIO, fal.ai 연동(채팅·이미지·TTS 일부) + 문서 RAG 색인.
- **인프라**: Docker Compose (postgres+redis+minio+api+worker+nginx), Nginx 리버스 프록시(8080).
- **크로스 플랫폼**: Makefile(macOS/Linux) + compose.ps1/compose.cmd(Windows)로 동일 워크플로우.
- **Studio API 보안**: django-ratelimit (엔드포인트별 IP 제한), 프롬프트·텍스트 길이 제한. 프로덕션에서 SECRET_KEY·MINIO 비밀번호 반드시 설정.
